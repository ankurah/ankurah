# 📦 Plan: Embedding Ankurah in React Native via UniFFI

This plan enables running an embedded Ankurah node inside a **React Native app**, with:
- Native performance (Rust + Sled storage)
- Auto-generated TypeScript bindings (via UniFFI + uniffi-bindgen-react-native)
- API parity with the existing WASM bindings where possible
- iOS + Android support

---

## Overview

### Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                     React Native App                        │
│                    (TypeScript/JSX)                         │
├─────────────────────────────────────────────────────────────┤
│              uniffi-bindgen-react-native                    │
│         (JSI Turbo Module - auto-generated)                 │
├─────────────────────────────────────────────────────────────┤
│     Swift (iOS) / Kotlin (Android) - auto-generated         │
├─────────────────────────────────────────────────────────────┤
│                    Rust FFI Layer                           │
│              (ankurah-rn-bindings crate)                    │
├─────────────────────────────────────────────────────────────┤
│                      Ankurah Core                           │
│        (Node, Context, Models, LiveQueries, etc.)           │
├──────────────────────┬──────────────────────────────────────┤
│   Sled Storage       │      WebSocket Client                │
│   (local DB)         │   (ankurah-websocket-client)         │
└──────────────────────┴──────────────────────────────────────┘
```

The Swift/Kotlin layers are auto-generated by the toolchain - you don't write or maintain them directly.

### Key Versions

- **uniffi-rs**: 0.29.x (matches uniffi-bindgen-react-native)
- **uniffi-bindgen-react-native**: 0.29.3-1 ([changelog](https://github.com/jhugman/uniffi-bindgen-react-native/blob/main/CHANGELOG.md))
- **React Native**: Latest stable (bare RN, not Expo initially)

## Phased Approach

### Phase 1: Proof of Concept (No Ankurah)

Verify the UniFFI + React Native toolchain works with async functions and callbacks.

1. Create `ankurah-react-native-template` repo
2. Set up bare React Native project
3. Create minimal UniFFI Rust crate with:
   - Async function returning a value
   - Callback-based subscription pattern
4. Wire together and verify on iOS simulator

### Phase 2: Ankurah Integration

Add Ankurah with local development patching (via `akdev` script).

1. Add `ankurah-rn-bindings` crate (similar to `wasm-bindings`)
2. Integrate:
   - `ankurah` with `uniffi` feature
   - `ankurah-storage-sled` for local storage
   - `ankurah-websocket-client` for server connectivity
3. Export core APIs: `Context`, `Transaction`, model operations

### Phase 3: Model Derive Macros

Add `#[cfg(feature = "uniffi")]` code generation to ankurah-derive.

1. Generate monomorphized wrappers (like WASM):
   - `MessageView`, `MessageMutable`, `MessageResultSet`, etc.
   - Static namespace methods: `Message.get()`, `Message.query()`, `Message.create()`
2. Maintain API parity with WASM bindings where possible

### Phase 4: Reactive UI & Full App Port

Port the complete chat application from `ankurah-react-sled-template`.

1. Research UniFFI callback patterns for reactive updates
2. Implement RN equivalents of `useObserve` / `signalObserver`
3. Copy server and model verbatim from `ankurah-react-sled-template`
4. Systematically port all React components to React Native
5. Adapt platform-specific features (localStorage → AsyncStorage, etc.)

```rust
use std::sync::Arc;

// Async function test
#[uniffi::export]
pub async fn greet_async(name: String) -> String {
    // Simulate async work
    tokio::time::sleep(std::time::Duration::from_millis(100)).await;
    format!("Hello, {}!", name)
}

// Callback test
#[uniffi::export(callback_interface)]
pub trait UpdateCallback: Send + Sync {
    fn on_update(&self, value: String);
}

#[derive(uniffi::Object)]
pub struct Counter {
    value: std::sync::atomic::AtomicU32,
    callback: Option<Arc<dyn UpdateCallback>>,
}

#[uniffi::export]
impl Counter {
    #[uniffi::constructor]
    pub fn new() -> Self {
        Self {
            value: std::sync::atomic::AtomicU32::new(0),
            callback: None,
        }
    }

    pub fn set_callback(&mut self, callback: Arc<dyn UpdateCallback>) {
        self.callback = Some(callback);
    }

    pub fn increment(&self) -> u32 {
        let new_val = self.value.fetch_add(1, std::sync::atomic::Ordering::SeqCst) + 1;
        if let Some(cb) = &self.callback {
            cb.on_update(format!("Counter is now: {}", new_val));
        }
        new_val
    }

    pub fn get(&self) -> u32 {
        self.value.load(std::sync::atomic::Ordering::SeqCst)
    }
}

uniffi::setup_scaffolding!();
```

#### `rn-bindings/build.rs`

```rust
fn main() {
    uniffi::generate_scaffolding("src/lib.rs").unwrap();
}
```

```bash
  --crate rn-bindings \
  --out-dir ../react-app/rust-bindings
```

Follow uniffi-bindgen-react-native documentation for:
- Adding the generated module to the RN project

### 2.3 Bindings Crate Structure

```rust
// rn-bindings/src/lib.rs
use std::sync::Arc;
use ankurah::{Node, PermissiveAgent};
use ankurah_storage_sled::SledStorageEngine;
use ankurah_websocket_client::WebsocketClient;

// Re-export model types (from shared model crate)
pub use ankurah_rn_model::*;

#[derive(uniffi::Object)]
pub struct AnkurahContext {
    node: Node<SledStorageEngine, PermissiveAgent>,
    client: Option<WebsocketClient>,
}

#[uniffi::export]
impl AnkurahContext {
    #[uniffi::constructor]
    pub async fn new(storage_path: String, server_url: Option<String>) -> Result<Self, String> {
        let storage = SledStorageEngine::new(&storage_path)
            .map_err(|e| e.to_string())?;
        let node = Node::new(Arc::new(storage), PermissiveAgent::new());
        
        let client = if let Some(url) = server_url {
            Some(WebsocketClient::new(node.clone(), &url)
                .await
                .map_err(|e| e.to_string())?)
        } else {
            None
        };
        
        node.system.wait_system_ready().await;
        
        Ok(Self { node, client })
    }
    
    // ... context methods
}

uniffi::setup_scaffolding!();
```

## Phase 4: Reactive UI & Full App Port

Port the complete chat application from `ankurah-react-sled-template` to React Native.

### 4.1 Research: Reactive Patterns

Before porting UI, evaluate UniFFI + RN reactive patterns:

1. How do UniFFI callbacks interact with React Native's JS thread?
2. Can we create a `useAnkurahSignal` hook that works similarly to `useObserve`?
3. What's the best pattern for LiveQuery subscriptions in RN?

### 4.2 Implement React Native Signal Hooks

Create RN equivalents of the WASM signal utilities:

| WASM (ankurah-react-sled-template) | React Native Equivalent |
|-----------------------------------|------------------------|
| `signalObserver(Component)` | `signalObserver(Component)` or similar HOC |
| `useObserve()` | `useObserve()` via UniFFI callbacks |
| `JsValueMut` / `JsValueRead` | RN-compatible reactive state wrapper |

```typescript
// Potential approach
function useAnkurahQuery<T>(queryFn: () => LiveQuery<T>) {
  const [items, setItems] = useState<T[]>([]);
  
  useEffect(() => {
    const liveQuery = queryFn();
    const unsubscribe = liveQuery.subscribe((changeset) => {
      setItems(changeset.resultset.items);
    });
    return unsubscribe;
  }, []);
  
  return items;
}
```

### 4.6 Port Components

Systematically port each component from `ankurah-react-sled-template/react-app/src/`:

#### Core App Structure
| File | RN Considerations |
|------|-------------------|
| `App.tsx` | Entry point - adapt for RN navigation if needed |
| `main.tsx` | RN uses `index.js` - different entry pattern |

#### Components (in dependency order)

1. **Header.tsx** / **Header.css**
   - Replace HTML elements with RN `View`, `Text`, `TouchableOpacity`
   - Convert CSS to StyleSheet

2. **RoomList.tsx** / **RoomList.css**
   - Use RN `FlatList` instead of mapped divs
   - Adapt click handlers to `onPress`

3. **MessageRow.tsx** / **MessageRow.css**
   - Core message display component
   - Convert to RN primitives

4. **MessageInput.tsx** / **MessageInput.css**
   - Use RN `TextInput`
   - Handle keyboard behavior (KeyboardAvoidingView)

5. **Chat.tsx** / **Chat.css**
   - Main chat view container
   - Use RN `FlatList` for message list with inverted scroll

6. **EditableTextField.tsx** / **EditableTextField.css**
   - Inline editing component
   - Adapt for RN text input patterns

7. **MessageContextMenu.tsx** / **MessageContextMenu.css**
   - Long-press context menu
   - Use RN `Modal` or ActionSheet

8. **ChatDebugHeader.tsx** / **ChatDebugHeader.css**
   - Debug info display
   - Straightforward conversion

9. **DebugOverlay.tsx**
   - Debug mode overlay
   - Convert to RN overlay pattern

10. **QRCodeModal.tsx** / **QRCodeModal.css**
    - QR code display for sharing
    - Use RN QR code library

#### Hooks
| File | Notes |
|------|-------|
| `hooks/useDebugMode.ts` | Should work as-is (standard React hook) |

#### Utilities
| File | Notes |
|------|-------|
| `ChatScrollManager.ts` | Adapt for RN FlatList scroll management |
| `NotificationManager.ts` | Use RN push notification APIs |

### 4.7 Platform-Specific Adaptations

| Web Feature | React Native Equivalent |
|-------------|------------------------|
| `localStorage` | `@react-native-async-storage/async-storage` |
| CSS files | `StyleSheet.create()` |
| `<div>`, `<span>` | `<View>`, `<Text>` |
| `<input>` | `<TextInput>` |
| `onClick` | `onPress` |
| `window.location` | RN doesn't have this - pass server URL as config |
| CSS hover states | Remove or use pressable states |
| Web notifications | RN push notifications |
